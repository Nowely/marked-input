## Your First Editor

Create a new React component with Markput:

```tsx
import { MarkedInput } from 'rc-marked-input'
import { useState } from 'react'

function MyEditor() {
  const [value, setValue] = useState('Hello @[World](meta)!')

  return (
    <MarkedInput
      value={value}
      onChange={setValue}
      Mark={(props) => <mark>{props.value}</mark>}
    />
  )
}

export default MyEditor
```

### How It Works

Let's break down what's happening:

1. **Value String**: `'Hello @[World](meta)!'` contains a markup pattern `@[value](meta)`
2. **MarkedInput Component**: Parses the string and identifies the pattern
3. **Mark Component**: Renders the annotated part - receives `props.value` ("World") and `props.meta` ("meta")

The text is automatically parsed, and anything matching the pattern becomes an interactive mark.

## Understanding Basic Concepts

### Markup Patterns

Markput uses **markup patterns** to identify which parts of text should be annotated. Think of them as templates:

- `@[__value__]` - Simple pattern (just a value)
- `@[__value__](__meta__)` - Pattern with metadata
- `**__value__**` - Markdown-style pattern

The `__value__` and `__meta__` placeholders tell Markput what to extract.

### The Mark Component

The Mark component receives props extracted from the pattern:

```tsx
const Mark = (props) => {
  // props.value - the text inside [...]
  // props.meta - the text inside (...)
  // props.children - nested content (for nested marks)
  return <span>{props.value}</span>
}
```

### Parsing Flow

1. User types: `Hello @[Alice](user:123)`
2. Markput parses and finds pattern `@[__value__](__meta__)`
3. Extracts: `value="Alice"`, `meta="user:123"`
4. Renders: `<Mark value="Alice" meta="user:123" />`

## Making it Interactive

You can use any React component as a Mark. Here's a clickable mention:

```tsx
import { MarkedInput } from 'rc-marked-input'
import { useState } from 'react'

function MyEditor() {
  const [value, setValue] = useState('Click on @[Alice](user:1)!')

  const ClickableMention = ({ value, meta }) => (
    <button
      onClick={() => alert(`User ID: ${meta}`)}
      style={{
        background: '#e3f2fd',
        border: '1px solid #2196f3',
        borderRadius: '4px',
        padding: '2px 6px',
        cursor: 'pointer',
      }}
    >
      @{value}
    </button>
  )

  return (
    <MarkedInput
      value={value}
      onChange={setValue}
      Mark={ClickableMention}
    />
  )
}
```

The Mark component can be styled, made interactive, and handle events just like any React component.

## Adding Autocomplete

Add a suggestion overlay that appears when typing a trigger character:

```tsx
import { MarkedInput } from 'rc-marked-input'
import { useState } from 'react'

function MyEditor() {
  const [value, setValue] = useState('Type @ to mention someone!')

  return (
    <MarkedInput
      value={value}
      onChange={setValue}
      Mark={(props) => <span style={{ color: 'blue' }}>@{props.value}</span>}
      options={[
        {
          markup: '@[__value__](user)',
          slotProps: {
            overlay: {
              trigger: '@',
              data: ['Alice', 'Bob', 'Charlie'],
            },
          },
        },
      ]}
    />
  )
}
```

When the user types `@`, a suggestion overlay appears. Selecting a name inserts `@[Alice](user)` into the text.

## Common Markup Patterns

| Pattern | Syntax | Example |
|---------|--------|---------|
| **Simple** | `@[__value__]` | `Hello @[World]!` |
| **With Meta** | `@[__value__](__meta__)` | `Hello @[World](id:1)!` |
| **Reversed** | `@(__value__)[__meta__]` | `Hello @(World)[id:1]!` |
| **Markdown** | `**__value__**` | `**Bold text**` |
| **Hashtag** | `#[__value__]` | `#[javascript]` |
| **Command** | `/[__value__]` | `/[help]` |

You can define any pattern that suits your use case!

## TypeScript Support

Define props for your Mark component for type safety:

```tsx
interface MyMarkProps {
  userId: string
  label: string
}

<MarkedInput<MyMarkProps>
  value={value}
  onChange={setValue}
  Mark={(props) => <span data-id={props.userId}>{props.label}</span>}
  options={[
    {
      markup: '@[__value__](__meta__)',
      // TypeScript will ensure correct prop types
    },
  ]}
/>
```
