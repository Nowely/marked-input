## How It Works

Understanding the core concepts:

| Concept | What it is | Example |
|---------|-----------|---------|
| **Markup Pattern** | Template defining how marks look in text | `@[__value__](__meta__)` |
| **Mark Component** | Your React component rendering the pattern | `<Mention user="Alice" />` |
| **Overlay** | Autocomplete menu triggered by a character | Dropdown when typing `@` |

### The Flow

1. User types: `Hello @[Alice](user:123)`
2. Markput parses the pattern `@[__value__](__meta__)`
3. Extracts: `value="Alice"`, `meta="user:123"`
4. Renders: `<Mark value="Alice" meta="user:123" />`

Your editor state is just a plain string:
```tsx
const [value, setValue] = useState('Hello @[Alice](user:1)!')
```

Easy to save, version, and work with any backend.

## Making Marks Interactive

Your Mark component can do anything a React component can:

```tsx
const Mention = ({ value, meta }) => (
  <button
    onClick={() => alert(`User ID: ${meta}`)}
    style={{
      background: '#e3f2fd',
      border: '1px solid #2196f3',
      borderRadius: '4px',
      padding: '2px 6px',
      cursor: 'pointer',
    }}
  >
    @{value}
  </button>
)

<MarkedInput
  value="Click on @[Alice](user:1)!"
  onChange={setValue}
  Mark={Mention}
/>
```

Clicks, hovers, custom styling - full React component control.

## Common Patterns

Markput works with any pattern you define:

| Pattern | Syntax | Use Case |
|---------|--------|----------|
| **Mentions** | `@[__value__](__meta__)` | Social features, user tags |
| **Hashtags** | `#[__value__]` | Categories, search |
| **Commands** | `/[__value__]` | Command palette |
| **Bold** | `**__value__**` | Text formatting |
| **Links** | `[__value__](__meta__)` | Hyperlinks with URLs |

See [all examples](../examples/mention-system) for complete implementations.

## TypeScript Support

Add type safety to your Mark components:

```tsx
interface MentionProps {
  value: string
  meta: string
}

<MarkedInput<MentionProps>
  value={value}
  onChange={setValue}
  Mark={(props) => {
    // props.value and props.meta are fully typed
    return <span data-id={props.meta}>@{props.value}</span>
  }}
/>
```
