## Your First Editor

Let's build a mention editor with autocomplete in three steps.

### Step 1: Render Marks

Create an editor that renders mentions. Click the mention to see it's interactive:

<CodePreview code={step1DemoRaw}>
  <Step1Demo client:visible />
</CodePreview>

**How it works:**

The text `@[World](user:123)` contains a **markup** that Markput recognizes and parses:

1. **Pattern**: `@[__value__](__meta__)` defines the structure
2. **Parsing**: Markput extracts `value="World"` and `meta="user:123"`
3. **Rendering**: Your `Mark` component receives these props and renders the mention

Since `Mark` is just a React component, you can add event handlers, styles, or any other React features. Your editor state remains a simple string—easy to save, version, and work with any backend:

```tsx
const [value, setValue] = useState('Hello @[World](user:123)!')
```

### Step 2: Add Autocomplete

Now add the `options` prop to show suggestions when typing `@`:

<CodePreview code={step2DemoRaw}>
  <Step2Demo client:visible />
</CodePreview>

**How it works:**

The `options` prop configures an **overlay**—an autocomplete menu that appears when you type a trigger character:

- **`markup`**: The pattern that will be inserted when user selects an item
- **`trigger`**: Character that opens the overlay (in this case `@`)
- **`data`**: Array of suggestions to show

When you type `@`, Markput automatically shows the built-in `Suggestions` component with filtered results. Selecting an item inserts the markup with the chosen value.

### Step 3: Custom Overlay

Create your own autocomplete UI using the `useOverlay` hook:

<CodePreview code={step3DemoRaw}>
  <Step3Demo client:visible />
</CodePreview>

**How it works:**

The `useOverlay` hook gives you full control over the overlay UI. It provides:

- **`match`**: Current search query and matched text information
- **`select`**: Function to insert the selected item with `{value, meta}`
- **`style`**: Positioning coordinates (`top`, `left`) for the overlay
- **`ref`**: Reference to attach to your overlay element

In this example, we fetch GitHub users and use the HTML Popover API for a native popup experience. The overlay automatically positions itself near the cursor and handles keyboard navigation.

## Common Patterns

Markput works with any pattern you define:

| Pattern | Syntax | Use Case |
|---------|--------|----------|
| **Mentions** | `@[__value__](__meta__)` | Social features, user tags |
| **Hashtags** | `#[__value__]` | Categories, search |
| **Commands** | `/[__value__]` | Command palette |
| **Bold** | `**__value__**` | Text formatting |
| **Links** | `[__value__](__meta__)` | Hyperlinks with URLs |

See [all examples](../examples/mention-system) for complete implementations.

## TypeScript Support

Add type safety to your Mark components:

```tsx
interface MentionProps {
  value: string
  meta: string
}

<MarkedInput<MentionProps>
  value={value}
  onChange={setValue}
  Mark={(props) => {
    // props.value and props.meta are fully typed
    return <span data-id={props.meta}>@{props.value}</span>
  }}
/>
```
