# 📊 Parser v1 vs Parser v2 - Baseline Performance Results

**Дата:** 4 октября 2025  
**Среда:** Node.js v20.16.0, macOS  
**Статус:** До оптимизации ParserV2

## 🚀 Ключевые результаты

### Производительность (ops/sec)
| Сценарий | Parser v1 | Parser v2 | Разница |
|----------|-----------|-----------|---------|
| Малый текст (10 маркеров) | 183,968 | 40,666 | **4.5x быстрее** |
| Средний текст (50 маркеров) | 34,436 | 8,000 | **4.3x быстрее** |
| Большой текст (100 маркеров) | 18,865 | 3,880 | **4.9x быстрее** |
| Социальная сеть | 689,415 | 152,199 | **4.5x быстрее** |
| Markdown | 1,480,045 | 291,085 | **5x быстрее** |
| **Сложная вложенность** | 741,170 | 14,005 | **52.9x быстрее** ⚠️ |

### Память (теоретические оценки)
| Сценарий | Parser v1 | Parser v2 | Разница |
|----------|-----------|-----------|---------|
| Малый текст | ~2KB | ~5KB | **2.5x больше** |
| Средний текст | ~8KB | ~22KB | **2.8x больше** |
| Большой текст | ~15KB | ~45KB | **3x больше** |

### Масштабируемость
- **Parser v1:** Линейная деградация, хорошая масштабируемость
- **Parser v2:** Нелинейная деградация, плохая масштабируемость
- **Оба:** 24x замедление при росте входа с 10 до 250 маркеров

## 🎯 Выводы

### ✅ Parser v1 (Победитель)
- **4-5x быстрее** во всех сценариях
- **2.5-3x меньше памяти**
- **Линейная масштабируемость**
- **Рекомендуется для 99% случаев**

### ❌ Parser v2 (Требует оптимизации)
- Неоправданно низкая производительность
- Высокое потребление памяти
- Плохая масштабируемость
- Подходит только для настоящей вложенности

## 🔧 План оптимизации ParserV2

1. **Оптимизировать построение дерева** для плоских данных
2. **Добавить fast-path** для невложенного контента
3. **Уменьшить object creation overhead**
4. **Реализовать lazy tree building**
5. **Оптимизировать memory allocation**
