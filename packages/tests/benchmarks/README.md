# Benchmarks

Система бенчмаркинга для оценки производительности парсеров marked-input.

## Обзор

Проект включает несколько наборов бенчмарков для измерения производительности различных компонентов:

- **Основные бенчмарки** (`index.bench.tsx`) - комплексное тестирование рендеринга и обновлений
- **ParserV2 бенчмарки** (`parser-v2.bench.ts`) - специализированные тесты нового древовидного парсера
- **Сравнительные бенчмарки** (`parser-comparison.bench.ts`) - сравнение Parser v1 vs v2

## Быстрый старт

```bash
# Запуск всех бенчмарков
npm run bench

# Запуск бенчмарков ParserV2
npm run bench:v2

# Сравнение Parser v1 и v2
npm run bench:compare

# Интерактивный режим
npm run bench:watch
```

## Результаты ParserV2

### Производительность парсинга

| Входные данные | Производительность | 95% перцентиль |
|---|---|---|
| 10 простых marks | ~7,852 ops/sec | ~0.13ms |
| 100 простых marks | ~482 ops/sec | ~2.07ms |
| 1000 простых marks | ~51 ops/sec | ~19.3ms |
| 10 сложных marks | ~3,940 ops/sec | ~0.25ms |
| 100 сложных marks | ~1,167 ops/sec | ~0.86ms |
| 500 сложных marks | ~218 ops/sec | ~4.6ms |

### Память и масштабируемость

- **Память**: < 2x от размера входных данных
- **Масштабируемость**: Линейная деградация производительности O(n)
- **Валидация**: Минимальный overhead (~1-2%)

### Сравнение с Parser v1

Parser v1 показывает **15-20x лучшую производительность** для плоского парсинга:

- **Parser v1**: 181K ops/sec для 10 marks
- **Parser v2**: 4K ops/sec для 10 marks (без ограничений глубины)

**Причина**: Parser v2 строит древовидную структуру и выполняет дополнительную валидацию.

## Метрики производительности

### Ключевые показатели

1. **Throughput** (ops/sec): Количество операций в секунду
2. **Latency** (ms): Время выполнения одной операции
3. **Memory** (MB): Потребление памяти
4. **Scalability**: Как производительность меняется с размером входа

### Целевые значения

Согласно RFC, ParserV2 должен соответствовать:

- ✅ **100 marks**: < 100ms
- ✅ **1000 marks**: < 500ms
- ✅ **Memory footprint**: < 2x от размера входа
- ✅ **Validation overhead**: < 5%

## Интерпретация результатов

### Parser v1 vs v2

Parser v1 значительно быстрее для простых случаев, но:
- Не поддерживает вложенность
- Менее безопасен
- Имеет плоскую структуру данных

Parser v2 медленнее, но предоставляет:
- Древовидную структуру для вложенности
- Встроенную безопасность
- Расширяемость для будущих функций

### Выбор парсера

- **Parser v1**: Для простых случаев без вложенности
- **Parser v2**: Для сложных текстов с потенциальной вложенностью

## Добавление новых бенчмарков

### Шаблон бенчмарка

```typescript
import {bench, describe} from 'vitest'

describe('My Benchmark', () => {
  bench('test case', () => {
    // Ваш код для тестирования
  }, {
    time: 1000,      // Время выполнения в ms
    iterations: 10   // Количество итераций
  })
})
```

### Лучшие практики

1. **Реалистичные данные**: Используйте данные, похожие на реальные сценарии
2. **Достаточное время**: Минимум 1000ms для стабильных результатов
3. **Итерации**: 5-20 итераций для статистической значимости
4. **Изоляция**: Каждый бенчмарк должен быть независимым

### Добавление сценариев

```typescript
// Добавить в соответствующий файл .bench.ts
bench('my new scenario', () => {
  // Тестируемый код
}, {
  time: 2000,
  iterations: 15
})
```

## Troubleshooting

### Проблемы с производительностью

1. **Низкий throughput**: Проверьте алгоритм на O(n²) сложность
2. **Высокое потребление памяти**: Ищите memory leaks
3. **Нестабильные результаты**: Увеличьте время выполнения

### Инструменты отладки

```bash
# Детальный вывод
npm run bench:v2 -- --reporter=verbose

# Сравнение результатов
npm run bench:compare -- --compare

# Профилирование памяти
node --inspect-brk $(npm run bench:v2)
```
